Java学习中零碎的知识点4

发表于 2020-04-08

这篇文章主要用来记录学习Java过程中一些比较零碎的知识点。

接口、lambda表达式与内部类

  1. 接口用来描述类应该做什么，而不指定他们应该如何做。一个类可以实现一个或多个接口。
  2. 接口中所有的方法都是public的，变量都是静态公有常量。
  3. Comparable接口、Cloneable接口。
  4. Java8中，允许在接口中增加了静态方法。Java9中接口中的方法可以是private，private方法可以是静态方法或者实例方法。也可以为接口方法提供一个默认实现，必须用default修饰符标记这样一个方法，默认方法的一个重要方法是“接口演化”。在接口中增加静态方法有违接口设计的初衷，所以现在的做法都是将静态方法放在伴随类中，如Collection/Collections、Path/Paths。
  5. 解决默认方法冲突：
    • 超类优先。
    • 接口冲突的时候，必须进行覆盖。
  6. 一种程序设计模式：回调。指定某个特定事件发生时应该采取的动作。
  7. 如果我们按照字典序进行排序的话，直接调用Array.sort即可，但当我们想按照长度或者其他方面进行比较的话，可以使用Array.sort的另一个版本，提供一个数组和比较器，比较器是实现了Comparator接口类的实例
  8. lambda表达式就是一个代码块，以及必须传入代码的变量规范。
    lambda表达式的形式
  9. 对于只有一个抽象方法的接口，需要这种接口的对象时，就可以提供一个lambda表达式。这种接口称为函数式接口。实际上，在Java中，对lambda表达式所能做的事情也就是转换为函数式接口。
    有时，lambda表达式涉及一个方法
    var timer = new Timer(1000, System.out::println);
    表达式System.out::println是一个方法引用，它指示编译器生成一个函数式接口的实例，覆盖这个接口的抽象方法给定的方法。
    注意只有当lambda表达式的体只调用一个方法而不做其他操作时，才能把lambda表达式重写为方法引用。
    关于代码块以及自由变量值有一个术语：闭包。
  10. 内部类的概念：内部类是定义在另一个类中的类。
    • 内部类可以对同一个包中的其他类隐藏。
    • 内部类方法可以访问定义这个类的作用域中的数据，包括原本私有的数据。
  11. 利用代理可以在运行时创建实现了一组给定接口的新类。Proxy类。
