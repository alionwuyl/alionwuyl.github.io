Java学习中零碎的知识点5

发表于 2020-04-08

这篇文章主要用来记录学习Java过程中一些比较零碎的知识点。

异常、断言和日志

  1. 在JAVA中，所有的异常对象都是派生于Throwable类的一个实例。Trowable类有两个子类，一个是ERROR，另一个是EXCEPTION，ERROR类层次结构描述了JAVA运行时系统的内部错误和资源耗尽错误。EXCEPTION层次也有两个分支，一个是RUNTIMEEXCEPTION，然后是其他异常。一般的规则是：由编程错误导致的异常属于RUNTIMEEXCEPTION，如果程序没有问题，但由于像I/O错误这类问题导致的异常属于其他异常。
    派生于RUNTIMEEXCEPTION的异常通常包含以下问题：
  • 错误的强制类型转换。
  • 数组访问越界。
  • 访问null指针。
    不是派生于RUNTIMEEXCEPTION的异常包括：
  • 驶入超越文件尾继续读取数据。
  • 试图打开一个不存在的文件。
  • 试图根据给定的字符串查找Class对象，而这个字符串表示得类不存在。
    即如果出现RUNTIMEEXCEPTION异常，那么一定是程序员的错误。
  1. 自定义异常通常要包含两个构造器，一个是默认的构造器，另一个是包含详细描述信息的构造器。
  2. 一般我们要捕获我们知道如何处理的异常，而继续传播那些不知道怎么处理的异常类。
  3. 如果一个类实现了AutoCloseable接口的类，可以使用try-with-resources语句
  4. 断言机制允许在测试期间向代码中插入一些检查，而在生产代码中会自动删除这些检查。
    一共有两种形式：
  • assert condition
  • assert condition : expression
    这两个语句都会计算条件，如果结果为false，这抛出一个AssertionERROR异常。在第二个语句中，表达式将传入AssertionError对象的构造器，并转换成一个消息字符串。
  1. 默认情况下断言是禁止的。在运行程序的时候用 -enableassertions或-ea选项启动断言，java -enableassertion MyApp,不必重新编译程序来启动或者禁用断言，启用或禁用断言是类加载器的功能。禁用断言的时候，类加载器会去除断言代码，因此不会降低程序运行的速度。
  2. 什么时候可以使用断言？
  • 断言失败是致命的、不可恢复的错误。
  • 断言检查只是在开发和测试阶段代开。
  1. 使用日志API的一些简单优点。
  • 可以很容易的取消全部日志记录，或者仅仅取消某个级别以下的日志，而且可以很容易的再次打开日志记录。
  • 可以很简单的禁止日志记录，因此，将这些日志代码留在程序中的开销很小。
  • 日志记录可以被定向到不同的处理器，如在控制台显示，写至文件等等。
  • 日志记录器和处理器都可以对记录进行过滤。过滤器可以根据过滤器实现指定的标准丢弃那些无用的记录项。
  • 日志记录可以采用不同的方式进行格式化，比如纯文本和XML。
  • 应用程序可以使用多个日志记录器，他们使用与包名类似的有层次结构的名字，比如com.mycompany.myapp。
  • 日志系统的配置由配置文件控制。
  1. 在java实验中尝试使用日志。
