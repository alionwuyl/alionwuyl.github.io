这篇文章主要用来记录学习Java过程中一些比较零碎的知识点。

集合

  1. Iterator接口包含四个方法：
  2. 为了让实现着更容易实现Collection接口，Java类库提供了一个类AbstractCollection，它保持size和iterator方法为抽象方法，实现了其他的例行方法。
  3. 集合类中的接口和类。

  4. List接口用于描述一个有序集合，并且集合中每个元素的位置很重要。有两种访问元素的协议：一种是通过迭代器，另一种是通过get和set方法随机的访问每个元素。
  5. 在java中，所有的链表实际都是双向链表的，即每个链接还存放着其前驱的引用。
  6. 如果一个迭代器发现它的集合被另一个迭代器修改了，或是被该集合自身的某个方法修改了，就会抛出ConcurrentModificationException异常。
    有一种简单的方法可以检测到并发修改，集合可以跟踪更改操作，每个迭代器为他所负责的更改操作维护一个单独的操作次数，在每个迭代器方法之前，比较两者即可。并发修改的检测有一个奇怪的例外，链表只跟踪结构性修改，set方法不被视为结构性修改。
  7. 在Java中，散列表用链表数组实现，每个列表被称为桶。在java8中，桶满时会从链表变为平衡二叉树。如果大致知道最终会有多少个元素插入到散列表中，就可以设置桶数。通常，将桶数设置为预计元素的75%~150%，标准类库使用的桶数是2的幂，默认值为16.装填因子可以确定何时对散列表进行再散列。例如，如果装填因子为0.75（默认值），说明表中已经填满了75%以上，会自动再散列，新表的桶数是原来的两倍，对于大多数应用来说，装填因子为0.75是合理的。
